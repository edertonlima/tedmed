Installation & upgrade - top

Block Double Logins requires WordPress v3.7 or higher to function properly!

Installation

  1. Download the ZIP package from CodeCanyon.
  2. Open WordPress admin and go to Plugins -> Add New -> Upload. Browse for the ZIP file block-double-logins.zip on your computer and click "Install Now".
  3. Activate the plugin.

Upgrade - method #1, it wont reset plugin's settings, log or reset current username/IP blocks

  1. Do NOT deactivate the plugin.
  2. Use FTP to locate folder wp-content/plugins/block-double-logins/
  3. Overwrite all files in that folder with new ones downloaded from CodeCanyon

Upgrade - method #2, it will reset plugin's settings, log and reset all username/IP blocks

  1. Deactivate and delete the plugin in your WP admin -> Plugins.
  2. Use the Add New -> Upload function to upload the new plugin ZIP file.
  3. Activate the plugin.

5sec getting started guide - top

Install and activate the plugin. All settings are available under Settings - Block Double Logins. Default settings will work for most sites; they prevent users from sharing accounts but IP restrictions are not enabled. Please click the Send URL to email for safekeeping button so that you have a way of going around the plugin's blocks in case you lock yourself out.

Current blocks are always visible in Settings, under the Tools tab.

Settings - top

All settings are available under Settings - Block Double Logins.

General Settings

Login Warning Message - message displayed on top of default login form. It serves as a warning to all who are logging in to the site. Leave blank to remove.
Plugin Override Code - use this code and URL to force your next login regardless of other people who may be logged in with the same IP or username. It disables the proxy block too. Please do not share this URL as it's supposed to be used only in emergencies. We also recommend using the "send to email" function so that you have the URL in a convenient location.
Session Lock Timeout - time for user inactivity after which the locks on username and IP are released. If the user properly logs out the lock is immediately released. Depending on the type of your site you should adjust the value to something lower (if you want the locks released faster) or higher (if there's no need for that). Default: 10 minutes.
Controlled User Roles - user roles on which all plugin rules are applied. To maximize security we recommend keeping the default setting - all roles; but adjust it to your needs as necessary.

Username Based Blocking

Prevent Double Login Based on Username - enables/disables the rule that at any given time a single WordPress account can only be used by one person.
Error Message on Login - message displayed on login form when double login is attempted.

IP Based Blocking

Prevent Double Login Based on IP - enables/disables the rule that at any given time a single IP can only be logged in with one WordPress account.
Error Message on Login - message displayed on login form when double login is attempted.
Block Users Who Use Proxies - people who use proxies will not be able to login. WARNING! There might be false negatives and in rare cases false positives. It's impossible to say with 100% certainty if someone is behind a proxy if they really want to hide it.
Error Message for Proxy Users - message displayed on login form after users behind a proxy attempt to login.

Tools

All double login attempts are logged and you can view the last 200 events in the log. You can also view current username and IP based locks. If needed those locks can be reset as well. That won't log you out. Just be sure to refresh the page after resetting so that you get a new session.

FAQ - top

What if I lock myself out?

Wait for 10 minutes (or the time you set for the session timeout) or use the plugin override code.

Will this work with any theme?

Yes, it's completely theme independant.

Is it compatible with other (security) plugins?

We haven't found any issues with other plugins, however we can't guarantee for "all" 3rd party plugins.

Will it slow my site down?

Plugin's overhead is absolutely minimal so it wont slow down your site.

Will it work with a custom login form?

Regardless of the login form that's used block will be applied only possible issue is that custom error messages shown on login won't be shown but that depends on the login form you're using.
